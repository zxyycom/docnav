# Docnav Risk Map

Use this reference to choose the smallest contract surface for a doubt cycle. Load only the sections that match the decision.

## Protocol and Schema

Check when the artifact touches raw protocol, schemas, examples, SDK-facing fields, or serialized output:

- Field names, required/optional status, nullability, enum values, and nesting.
- Error object shape, request id propagation, status fields, and pagination metadata.
- Compatibility between raw protocol and readable CLI output.
- `docs/schemas/` and `docs/examples/` updates when fields or example meaning changes.

Evidence sources: `docs/navigation.md` role path, `docs/protocol.md`, `docs/schemas/`, `docs/examples/`, schema validation, targeted tests.

## Ref and Pagination

Check when the artifact touches refs, continuation, page boundaries, ordering, limits, or range reads:

- Refs remain adapter-owned, generated by adapters, and passed through unchanged by core/MCP.
- Existing saved refs keep working unless a migration explicitly changes that contract.
- Continuation tokens and pagination metadata preserve deterministic ordering and bounded output.
- Limit semantics are documented and tested at the layer that owns them.

Evidence sources: `docs/refs.md`, `docs/adapter-contract.md`, adapter tests, saved fixture refs, CLI/MCP read flows.

## Adapter Contract

Check when the artifact changes outline/read/find behavior, detection, parsing, or adapter direct CLI output:

- Adapter owns format-specific parsing and navigation strategy.
- Core routes to adapters without duplicating format logic.
- Direct adapter CLI and core-mediated CLI remain compatible where promised.
- Error categories and unsupported-format behavior map correctly.

Evidence sources: `docs/adapter-contract.md`, CodeGraph callers/callees, adapter tests, direct adapter commands.

## CLI and MCP Output

Check when user-visible CLI text, raw/readable modes, MCP tool mapping, or error mapping changes:

- `docnav` core owns output mode and error mapping.
- `docnav-mcp` maps tool calls to `docnav` instead of reimplementing parsing or routing.
- Readable CLI output keeps information density without changing raw protocol semantics.
- MCP output does not introduce wrapper differences that consumers observe as contract drift.

Evidence sources: `docs/cli.md`, `docs/protocol.md`, MCP bridge tests, CLI golden outputs, command output.

## Security and External Boundaries

Check when the artifact touches untrusted documents, filesystem paths, process execution, environment variables, or generated output:

- Path handling keeps workspace and user-provided paths distinct.
- External commands have explicit arguments and bounded output.
- Malformed or hostile documents fail with controlled errors.
- Secrets, environment values, and local absolute paths are not exposed in public examples or logs.

Evidence sources: security-sensitive implementation surface, tests with malformed inputs, command invocation review, docs for public examples.

## Migration and Compatibility

Check when persisted data, saved refs, config files, examples, or downstream consumers can be affected:

- Existing users have a clear compatibility story.
- Breaking changes are explicit and paired with migration or versioning.
- Rollback/downgrade behavior is understood when relevant.
- Tests cover old and new behavior at the narrowest contract boundary.
